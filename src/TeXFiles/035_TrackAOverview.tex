This chapter outlines Track A: workflow automation and infrastructure R\&D. Track A is an independent engineering effort focused on integrating AI agent capabilities into existing systems and evaluating serving patterns.

%=============================================================================
\section{Goals and Scope}
\label{sec:TrackA_Goals}
%=============================================================================

Track A addressed four areas: (i) AI agent integration for ERP processes using n8n, (ii) evaluation of local and enterprise model serving approaches, (iii) RAG deployment for product question answering, and (iv) investigation of MCP as a standardized interface for tools and context interoperability.

%=============================================================================
\section{Components}
\label{sec:TrackA_Components}
%=============================================================================

Track A components interact as follows:

\begin{enumerate}
      \item \textbf{Enterprise System Layer}: ERP systems act as event sources, triggering workflows through API calls or event notifications.
      \item \textbf{n8n Orchestration}: Receives events from enterprise systems, invokes AI agents via MCP or direct API calls, handles retries and error recovery.
      \item \textbf{AI Layer}: vLLM instances process requests, with MCP Client handling tool invocation and context management.
      \item \textbf{Model Context Protocol}: Exposes standardized interfaces for tools and resources, enabling loose coupling between n8n and AI services.
\end{enumerate}
