This chapter describes the production architecture of the AgileTest Agent
platform built with LangGraph. The architecture shifts from exploratory
prototypes toward an industrial-grade system with deterministic behavior,
human-in-the-loop control, and operational robustness.

The agent is not a monolithic component but a collection of loosely coupled
services with defined responsibilities. This separation enables independent
scaling, governance, and observability, following cloud-native design
principles \cite{googleSRE,microsoftWellArchitected}.

%=============================================================================
\section{Component Architecture}
\label{sec:ComponentArchitecture}
%=============================================================================

The LangGraph platform comprises five components:

\begin{itemize}
    \item \textbf{LangGraph API service}: Core execution engine exposing HTTP endpoints for invocation, interrupt handling, and resume operations. Graph definitions load from a registry and execute as state machines.\smallskip
    \item \textbf{Persistence}: PostgreSQL stores graph threads and checkpoints (pause-and-resume); Redis handles operational state (token counters, consumption limits).\smallskip
    \item \textbf{Language model provider}: Abstracted interface enabling model substitution without graph redesign.\smallskip
    \item \textbf{Observability}: Langfuse callbacks record traces, latency, and metrics via best-effort tracing that does not interrupt execution.\smallskip
    \item \textbf{External clients}: Web applications and services interact only via the API layer.
\end{itemize}

This separation enables independent scaling, governance, and observability following cloud-native design principles.

\section{Deployment Architecture (Cloud Run)}
\label{sec:DeploymentArchitecture}
%=============================================================================

The platform is deployed on Google Cloud Run for automatic scaling and request-based billing. Two services run independently: primary LangGraph API for request execution, and secondary cron service for maintenance. Connection pooling to Redis is critical under concurrent load. Secrets injected via platform; network access restricted. Langfuse observability on GCP behind VPN.

\section{Runtime Request Lifecycle}
\label{sec:RuntimeLifecycle}
%=============================================================================

Requests are associated with a thread identifier representing execution context. At interrupt points, state is checkpointed and the API signals human input is needed. On resume, execution continues from the checkpoint. Callbacks record metrics and traces; governance checks enforce token limits before model calls. See component and control-point diagrams above.

\chapter{Graph Design and Runtime Behavior}
\label{ch:GraphDesign}
%=============================================================================
% Page budget: 7 pages (Pages 37--43)

This chapter covers the internal design of LangGraph graphs used by the
AgileTest Agent. Unlike traditional workflows, these graphs explicitly model
state, transitions, and validation boundaries, enabling controlled agent
behavior over unconstrained autonomy.

%=============================================================================
\section{State and Schema Design}
\label{sec:StateSchemaDesign}
%=============================================================================

Each graph operates over a strongly typed state object. Reducers define explicit state updates across node transitions. Language model outputs are constrained using structured schemas (Figures~\ref{fig:state_schema} and~\ref{fig:structured_output}), reducing ambiguity and enabling reliable downstream processing. Schema validation failures trigger retries with stricter prompts or safe abort paths rather than propagating invalid state.

\section{HITL Interrupt and Resume Lifecycle}
\label{sec:HITLLifecycle}
%=============================================================================

At human-in-the-loop interrupts (Figure~\ref{fig:hitl_timeline}), execution state is checkpointed with a thread identifier. Only validated state persists. On resume, user input is validated before merging into state, then execution continues deterministically from the interrupt point. This enables reliable human-agent collaboration while preserving execution guarantees.
