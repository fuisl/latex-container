\chapter{Methodology}
\chaptermark{Methodology}
\label{ch:Methodology}

The proposed framework, referred to as \emph{FGS} (FRAP-Graph-SAC), extends the multi-agent Soft Actor-Critic algorithm with graph-based inter-agent communication and phase-aware local encoding. Its second-generation variant, \emph{FGSv2}, replaces the single node embedding produced by the local encoder with a set of per-action tokens, enabling richer action-conditioned reasoning at both the actor and the centralized critic. Both variants follow the \emph{centralised training with decentralised execution} (CTDE) paradigm \citep{lowe2017multi}: at execution time each agent acts solely on its own local observation, while training exploits a centralized critic that has access to the joint state of all agents.

\section{Problem Formulation}
\label{sec:Methodology_ProblemFormulation}

A signalised road network with $N$ intersections is modelled as a Decentralized Partially Observable Markov Decision Process (Dec-POMDP) $\langle \mathcal{N}, \mathcal{S}, \{\mathcal{O}_i\}, \{\mathcal{A}_i\}, T, \{r_i\}, \gamma \rangle$, where $\mathcal{N} = \{1, \ldots, N\}$ is the set of agents, $\mathcal{S}$ is the global state of the SUMO environment, $\mathcal{O}_i$ is the local observation space of agent $i$, $\mathcal{A}_i$ is its action space, $T$ is the stochastic transition function determined by the simulator, $r_i$ is the local reward function, and $\gamma \in (0,1)$ is the discount factor. Agents act simultaneously at each decision step; each agent observes only the state of its own intersection and its immediate neighbourhood.

\section{Observation Space}
\label{sec:Methodology_ObservationSpace}

Each agent receives a dictionary observation at every decision step. The dictionary encodes the agent's local traffic state together with the graph structure of the surrounding network, allowing the model to perform graph message passing during the forward pass.

The node feature matrix $\mathbf{X} \in \mathbb{R}^{N \times d}$ stacks the feature vector of every agent in the network. For agent $i$, the feature vector is
\begin{equation}
  \mathbf{x}_i =
    \bigl[\,\mathbf{e}_{\mathrm{phase}}^{(i)},\;
           t_{\mathrm{green}}^{(i)},\;
           \boldsymbol{\rho}^{(i)},\;
           \mathbf{q}^{(i)}\,\bigr] \in \mathbb{R}^{d},
  \label{eq:node_feature}
\end{equation}
where $\mathbf{e}_{\mathrm{phase}}^{(i)}$ is a one-hot vector encoding the current active phase, $t_{\mathrm{green}}^{(i)}$ is the time elapsed since the last phase change normalised by the minimum green time, and $\boldsymbol{\rho}^{(i)}, \mathbf{q}^{(i)}$ are the lane density and lane queue vectors over all incoming lanes of intersection $i$.

The remaining fields in the observation dictionary are:
\begin{itemize}
  \item \textit{Edge index} $\mathcal{E} \in \mathbb{Z}^{2 \times |\mathcal{E}|}$: source--target integer pairs that define the agent communication graph.
  \item \textit{Edge mask} and \textit{edge weight}: a binary validity flag and a scalar weight $w_{ij}$ for each edge (see Section~\ref{sec:Methodology_DataPreprocessing}).
  \item \textit{Ego index}: the integer identifying which row of $\mathbf{X}$ corresponds to the acting agent.
  \item \textit{Action mask}: a binary vector of length $A_{\max}$ indicating the phases that are valid for the ego agent.
  \item \textit{Phase-pair mask} $\mathbf{M}_{\mathrm{pair}} \in \{0,1\}^{N \times A_{\max} \times M}$: for each agent and candidate phase, indicates the subset of traffic movements that phase controls.
  \item \textit{Phase-competition mask} $\mathbf{M}_{\mathrm{comp}} \in \{0,1\}^{N \times A_{\max} \times (A_{\max}-1)}$: encodes movement conflicts between phase $a$ and each of its $A_{\max}-1$ competitors; the self-competition entry is absent by construction.
  \item \textit{Previous joint action} $\mathbf{a}_{t-1} \in \{0,1\}^{N \times A_{\max}}$: one-hot encoding of all agents' actions at the preceding step.
\end{itemize}

\section{Action Space}
\label{sec:Methodology_ActionSpace}

The action space of each agent is discrete: at each step the agent selects a phase index from the set of phases its intersection supports. Because intersections in the German RESCO scenarios have different numbers of phases, the action space is padded to a uniform width $A_{\max} = \max_{i \in \mathcal{N}} |\mathcal{A}_i|$. During the forward pass, the logits of all invalid phases are masked to $-\infty$ before the softmax, ensuring that the sampled action always lies within the agent's valid phase set.

Phase transition constraints (minimum green time, yellow interval) are enforced at the SUMO environment level and are not modelled as RL constraints.

\section{Graph Construction}
\label{sec:Methodology_DataPreprocessing}

The agent communication graph $\mathcal{G} = (\mathcal{V}, \mathcal{E}, \mathbf{W})$ is derived from the SUMO road network file (\texttt{.net.xml}) before training begins and is fixed throughout.

\paragraph{Node extraction.}
Every junction that carries a traffic light logic (\texttt{tlLogic}) in the network XML is added to $\mathcal{V}$ as a graph node. Its geographic coordinates are recorded for visualisation but are not used as model inputs.

\paragraph{Super-edge construction.}
For each outgoing road segment from a traffic light node, a Dijkstra traversal follows legal road transitions until it reaches the nearest downstream traffic light node, accumulating the intermediate edges along the path. This yields a set of \emph{super-edges} that connect pairs of signalised intersections directly, abstracting away unsignalised junctions in between.

\paragraph{Edge weight assignment.}
The weight of super-edge $(i, j)$ is
\begin{equation}
  w_{ij} = \frac{1}{\tau_{ij}},
  \label{eq:edge_weight}
\end{equation}
where $\tau_{ij}$ is the estimated travel time along the path. When multiple parallel routes connect the same signal pair, their weights accumulate additively. All edges are inserted bidirectionally to enable symmetric message passing.

The resulting graph is serialised to disk as a pre-processing artefact and loaded once per training run, adding no per-step overhead.

\section{Baseline Algorithms}
\label{sec:Methodology_ModelsAlgorithmsFrameworks}

The following algorithms serve as baselines or as the algorithmic foundation of the proposed framework.

\subsection{Deep Q-Network}
\label{sec:Methodology_DQN}

Deep Q-Network (DQN) \citep{mnih2015humanlevel} approximates the action-value function with a neural network $Q_\theta(s, a)$ trained to minimise the temporal-difference loss
\begin{equation}
  \mathcal{L}(\theta) =
    \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}}
    \!\left[\left(r + \gamma \max_{a'} Q_{\bar\theta}(s', a') - Q_\theta(s, a)\right)^2\right],
\end{equation}
where $\mathcal{D}$ is a replay buffer and $\bar\theta$ are the parameters of a periodically updated target network. Each intersection runs an independent DQN with no inter-agent communication, forming a fully decentralised value-based baseline.

\subsection{Proximal Policy Optimisation}
\label{sec:Methodology_PPO}

Proximal Policy Optimisation (PPO) \citep{schulman2017proximal} is an on-policy actor-critic method that limits policy update size through a clipped surrogate objective:
\begin{equation}
  \mathcal{L}^{\mathrm{CLIP}}(\theta) =
    \mathbb{E}_t\!\left[
      \min\!\left(
        r_t(\theta)\,\hat{A}_t,\;
        \mathrm{clip}\!\left(r_t(\theta),\,1-\varepsilon,\,1+\varepsilon\right)\hat{A}_t
      \right)
    \right],
\end{equation}
where $r_t(\theta) = \pi_\theta(a_t|s_t) / \pi_{\theta_{\mathrm{old}}}(a_t|s_t)$ is the probability ratio and $\hat{A}_t$ is an advantage estimate. PPO is used as an independent-agent on-policy baseline.

\subsection{Soft Actor-Critic}
\label{sec:Methodology_SAC}

Soft Actor-Critic (SAC) \citep{haarnoja2018soft} augments the expected return objective with an entropy regularisation term:
\begin{equation}
  \pi^* = \arg\max_\pi \mathbb{E}\!\left[
    \sum_{t=0}^{T} \gamma^t
    \bigl(r_t + \alpha\,\mathcal{H}(\pi(\cdot|s_t))\bigr)
  \right],
  \label{eq:sac_objective}
\end{equation}
where $\alpha > 0$ is a temperature parameter balancing exploitation and exploration. Because traffic signal control admits only discrete phase actions, this work applies the discrete-action SAC formulation \citep{christodoulou2019soft}, which marginalises analytically over all actions rather than relying on the reparameterisation trick. Twin Q-networks $Q_{\theta_1}, Q_{\theta_2}$ are maintained to reduce overestimation bias. The discrete-action soft target value is
\begin{equation}
  V_{\mathrm{target}}(s') =
    \sum_{a'} \pi_\phi(a' \mid s')
    \!\left[
      \min_{k=1,2} Q_{\bar{\theta}_k}(s', a') - \alpha\log\pi_\phi(a' \mid s')
    \right],
  \label{eq:sac_target}
\end{equation}
\begin{equation}
  y = r + \gamma\,(1-d)\,V_{\mathrm{target}}(s'),
  \label{eq:sac_td_target}
\end{equation}
where $d \in \{0,1\}$ is the episode-termination flag. The critic and actor losses are, respectively:
\begin{align}
  \mathcal{L}_Q(\theta) &=
    \mathbb{E}_{(s,a,r,s')\sim\mathcal{D}}\!\left[
      \bigl(Q_\theta(s,a) - y\bigr)^2
    \right],
  \label{eq:sac_critic_loss}\\
  \mathcal{L}_\pi(\phi) &=
    \mathbb{E}_{s\sim\mathcal{D}}\!\left[
      \sum_{a}\pi_\phi(a \mid s)
      \Bigl(\alpha\log\pi_\phi(a \mid s) - \min_{k} Q_{\theta_k}(s,a)\Bigr)
    \right].
  \label{eq:sac_actor_loss}
\end{align}
The temperature $\alpha$ is itself optimised by minimising the dual objective
\begin{equation}
  \mathcal{L}(\alpha) =
    \mathbb{E}_{s\sim\mathcal{D}}\!\left[
      \alpha\,\bigl(\mathcal{H}(\pi(\cdot \mid s)) - \mathcal{H}_{\mathrm{target}}\bigr)
    \right],
  \label{eq:sac_alpha_loss}
\end{equation}
where $\mathcal{H}_{\mathrm{target}} = -\!\log(1/A_{\mathrm{valid}})$ is the entropy of the uniform policy over valid actions. Gradient derivations for all three SAC losses and the log-parameterisation of $\alpha$ are given in Appendix~\ref{sec:Appendix_SAC_gradients}. The off-policy design of SAC enables training from a prioritised replay buffer, reducing the number of environment interactions needed to reach convergence---an important property when SUMO simulations are the bottleneck.

\subsection{Graph Attention Layer}
\label{sec:Methodology_GAT}

Both FGS and FGSv2 use GATv2 \citep{brody2022how} as the graph communication operator. For context, the original CoLight method \citep{wei2019colight} employed a scaled dot-product attention mechanism in which the coefficient weighting the message from neighbour $j$ to ego $i$ is
\begin{equation}
  e_{ij} =
    \frac{
      \exp\!\left(\mathbf{p}_i^\top \mathbf{p}_j / \sqrt{h_d}\right)
    }{
      \sum_{k \in \mathcal{N}(i)} \exp\!\left(\mathbf{p}_i^\top \mathbf{p}_k / \sqrt{h_d}\right)
    },
  \label{eq:colight_attn}
\end{equation}
where $\mathbf{p}_i$ are projected representations of head dimension $h_d$ and $\mathcal{N}(i)$ is the neighbourhood of $i$ in $\mathcal{G}$. This dot-product form cannot vary its attention coefficients based on the query node independently of the key, a limitation identified by \citet{brody2022how}.

GATv2 corrects this with an additive attention formulation that is strictly more expressive:
\begin{equation}
  \alpha_{ij} =
    \frac{
      \exp\!\left(\mathbf{a}^\top \mathrm{LeakyReLU}\!\left(\mathbf{W}\,[\mathbf{h}_i \,\Vert\, \mathbf{h}_j]\right)\right)
    }{
      \sum_{k \in \mathcal{N}(i)}
      \exp\!\left(\mathbf{a}^\top \mathrm{LeakyReLU}\!\left(\mathbf{W}\,[\mathbf{h}_i \,\Vert\, \mathbf{h}_k]\right)\right)
    },
  \label{eq:gatv2_attn}
\end{equation}
where $\mathbf{W}$ and $\mathbf{a}$ are learnable parameters, $\Vert$ denotes concatenation, and $\alpha_{ij}$ is the normalised attention coefficient for edge $(i,j)$ — note that $\alpha_{ij}$ here is a graph-attention weight, distinct from the SAC temperature scalar $\alpha$ used in Section~\ref{sec:Methodology_SAC}. By applying the non-linearity before the inner product, GATv2 allows attention coefficients to depend jointly on both the query $\mathbf{h}_i$ and the key $\mathbf{h}_j$. Multi-head attention is applied with $H$ heads; outputs are concatenated and projected to the output dimension. All FGS and FGSv2 variants with graph communication use GATv2 as specified by Equation~\eqref{eq:gatv2_attn}. A proof of node-permutation equivariance and a discussion of edge-weight integration are in Appendix~\ref{sec:Appendix_GATv2_properties}.

\subsection{FRAP Encoder}
\label{sec:Methodology_FRAP}

FRAP \citep{zheng2019learning} encodes the competition between candidate signal phases by explicitly comparing each phase against every competing phase and pooling the competition results. The following description adopts the notation used by FGS and FGSv2; specifically, the hidden competition feature $\mathbf{c}_{i,a,b}$ produced in Step~3 is the tensor at which FGSv2 taps the encoder (Section~\ref{sec:Methodology_FGSv2}).

Let intersection $i$ have $M_i$ incoming traffic movements and $A_i$ candidate phases. The phase-pair mask $\mathbf{P}_i \in \{0,1\}^{A_i \times M_i}$ sets $P_{i,a,u}=1$ if movement $u$ is served by phase $a$. The phase-competition mask $\mathbf{M}_{\mathrm{comp}} \in \{0,1,2,3\}^{A_i \times (A_i-1)}$ encodes one of four predefined relation types between each ordered pair $(a, b)$ of distinct phases; type~0 indicates no movement conflict.

\paragraph{Step 1: Movement demand embedding.}
For each movement $u$, a phase indicator $p_{i,u}\in\{0,1\}$ (whether $u$ is currently green) and a demand scalar $d_{i,u}$ (lane queue length or vehicle density) are embedded separately, then fused through a shared lane-embedding layer:
\begin{equation}
  \mathbf{e}_{i,u} =
    \operatorname{ReLU}\!\Bigl(
      W_{\text{lane}}\bigl[
        \sigma\!\bigl(\operatorname{Emb}_{\text{phase}}(p_{i,u})\bigr),\;
        \sigma(W_d\,d_{i,u})
      \bigr]
    \Bigr) \in \mathbb{R}^{C},
  \label{eq:frap_movement_emb}
\end{equation}
where $\operatorname{Emb}_{\text{phase}}$ is a learnable phase embedding, $W_d$ is a demand projection, and $\sigma$ is the sigmoid function.

\paragraph{Step 2: Phase-pair embedding.}
For each candidate phase $a$, the phase-pair embedding aggregates the movement embeddings of the movements it controls via a scaled sum:
\begin{equation}
  \boldsymbol{\phi}_{i,a} = \sum_{u} w_{i,a,u}\,\mathbf{e}_{i,u} \in \mathbb{R}^{C},
  \label{eq:frap_phase_emb}
\end{equation}
where the weight $w_{i,a,u} = P_{i,a,u}\cdot\min(\sum_v P_{i,a,v},\,2)\,/\,\max(\sum_v P_{i,a,v},\,1)$ normalises each phase's contribution to the expected two-movement sum, ensuring that phases controlling different numbers of movements are represented on a comparable scale.

\paragraph{Step 3: Pairwise competition feature.}
For each ordered competitor pair $(a, b)$ with $b \ne a$, a hidden competition feature $\mathbf{c}_{i,a,b}$ is computed in three sub-steps. First, the concatenated phase-pair embeddings pass through a $1{\times}1$ convolutional layer to produce a latent interaction:
\begin{equation}
  \mathbf{h}_{i,a,b} =
    \operatorname{ReLU}\!\bigl(
      \operatorname{Conv}_{\text{phase}}\bigl([\boldsymbol{\phi}_{i,a} \;\Vert\; \boldsymbol{\phi}_{i,b}]\bigr)
    \bigr) \in \mathbb{R}^{C}.
  \label{eq:frap_h}
\end{equation}
Second, the relation type $R_{i,a,b}$ is projected to a relation embedding:
\begin{equation}
  \mathbf{r}_{i,a,b} =
    \operatorname{ReLU}\!\bigl(
      \operatorname{Conv}_{\text{rel}}\bigl(\operatorname{Emb}_{\text{rel}}(R_{i,a,b})\bigr)
    \bigr) \in \mathbb{R}^{C}.
  \label{eq:frap_r}
\end{equation}
Third, the interaction and relation are fused element-wise and passed through a second $1{\times}1$ convolution:
\begin{equation}
  \mathbf{c}_{i,a,b} =
    \operatorname{ReLU}\!\bigl(
      \operatorname{Conv}_{\text{hid}}\bigl(\mathbf{h}_{i,a,b} \odot \mathbf{r}_{i,a,b}\bigr)
    \bigr) \in \mathbb{R}^{C}.
  \label{eq:frap_c}
\end{equation}
The tensor $\mathbf{c}_{i,a,b}$ is the \emph{phase-competition representation} — a rich, per-competitor feature that captures both the relative demand between two phases and the structural movement-conflict type between them.

\paragraph{Step 4: Phase score (original FRAP output).}
In the original FRAP, a final $1{\times}1$ convolution projects each competition feature to a scalar:
\begin{equation}
  s_{i,a,b} = \operatorname{Conv}_{1\times1}(\mathbf{c}_{i,a,b}) \in \mathbb{R}.
  \label{eq:frap_scalar}
\end{equation}
The phase score for candidate phase $a$ is obtained by summing over all valid competitors $V_{i,a} = \{b \mid b\ne a,\, m_{i,b}=1\}$:
\begin{equation}
  f_i(a) = \sum_{b \in V_{i,a}} s_{i,a,b},
  \label{eq:frap_score}
\end{equation}
yielding one scalar per candidate phase that directly serves as a Q-value estimate in the original FRAP policy.

FGSv2 \emph{replaces} Steps 4a--4b entirely: instead of projecting $\mathbf{c}_{i,a,b}$ to a scalar, it mean-pools $\mathbf{c}_{i,a,b}$ across valid competitors to produce a rich action token $\mathbf{z}_{i,a}$, retaining the $C$-dimensional feature vector for downstream graph communication and action-conditioned actor computation (Equation~\ref{eq:fgsv2_token}).

The overall design is permutation-equivariant with respect to phase ordering: reordering the candidate phases produces a correspondingly reordered output vector, which supports generalisation across intersections with heterogeneous phase counts and movement configurations.

\section{Reward Function}
\label{sec:Methodology_Rewards}

\subsection{Differential Waiting Time}
\label{sec:Methodology_DiffWaitingTime}

All experiments use the differential waiting time reward:
\begin{equation}
  r_t = W_{t-1} - W_t,
  \qquad
  W_t = \frac{1}{100}\sum_{l \in \mathcal{L}}\sum_{v \in l} w_v(t),
  \label{eq:reward}
\end{equation}
where $\mathcal{L}$ is the set of incoming lanes controlled by the traffic signal, $v$ indexes vehicles, and $w_v(t)$ is the accumulated waiting time of vehicle $v$ at step $t$ as reported by SUMO via the TraCI interface. A positive value of $r_t$ indicates that total waiting time at the intersection decreased relative to the previous step. Dividing by 100 normalises the scale across scenarios that differ in traffic volume. Appendix~\ref{sec:Appendix_reward_telescope} proves that maximising the discounted sum of this reward is equivalent to minimising a discounted sum of waiting times.

This reward is local to each agent, which aligns with the CTDE assumption: the reward does not require knowledge of the global joint state at execution time, so decentralised policies remain valid after training.

\subsection{Alternative Reward Signals}
\label{sec:Methodology_AltRewards}

\paragraph{Delay.}
Delay quantifies the additional travel time incurred by vehicles relative to their free-flow speed. It captures network-level throughput effects directly but requires per-vehicle free-flow speed estimates from SUMO, increasing per-step computational cost. Delay is used as an \emph{evaluation metric} (Section~\ref{sec:ExperimentalDesign_MetricsProcedure}) rather than a training signal.

\paragraph{Pressure.}
Pressure \citep{wei2019presslight} measures the lane imbalance at an intersection:
\begin{equation}
  p_i = \left|\,\sum_{l \in \mathcal{L}_{\mathrm{in}}^{i}} \rho_l
              - \sum_{l \in \mathcal{L}_{\mathrm{out}}^{i}} \rho_l\,\right|,
\end{equation}
where $\rho_l$ is vehicle density on lane $l$. The pressure reward $r_t = -p_i(t)$ penalises queue accumulation and has been shown to stabilise single-agent training \citep{wei2019presslight}. It was evaluated as an alternative signal but not adopted as the primary reward in this work.

\subsection{FGS: Graph-Aware Multi-Agent SAC}
\label{sec:Methodology_FGS}

FGS wraps the SAC algorithm with a shared graph encoder that enables each agent to condition its policy on the current state of its neighbours. All agents share a single set of parameters, so the framework scales to networks of arbitrary size without any per-agent parameter budget.

\paragraph{Graph encoder.}
The graph encoder $f_\psi$ transforms the dictionary observation into an ego embedding used by the actor and a full graph tensor used by the centralized critic. Given node features $\mathbf{X} \in \mathbb{R}^{N \times d}$, the encoder proceeds as follows:
\begin{enumerate}
  \item \textit{Local encoding.} The feature matrix is unfolded to a batch of $N$ vectors. A local encoder $f_{\mathrm{enc}}$ maps each $\mathbf{x}_i$ to a local embedding $\mathbf{h}_i \in \mathbb{R}^{d_h}$:
  \[
    \mathbf{h}_i = f_{\mathrm{enc}}(\mathbf{x}_i), \quad i = 1,\ldots,N.
  \]
  Two choices of $f_{\mathrm{enc}}$ are evaluated: a two-layer MLP (FGS-MLP) and the FRAP encoder described in Section~\ref{sec:Methodology_FRAP} (FGS-FRAP).
  \item \textit{Graph communication (optional).} A single GAT or GATv2 layer aggregates messages from neighbours:
  \[
    \mathbf{h}_i^{\prime} = \mathrm{GAT}\bigl(\mathbf{h}_i,\; \{\mathbf{h}_j : j \in \mathcal{N}(i)\},\; \{w_{ij}\}\bigr).
  \]
  When communication is disabled, $\mathbf{h}_i^{\prime} = \mathbf{h}_i$.
  \item \textit{Output.} The graph tensor $\mathbf{H}^{\prime} \in \mathbb{R}^{N \times d_h^{\prime}}$ stacks all updated embeddings; the ego embedding $\mathbf{h}_{\mathrm{ego}}^{\prime}$ is the row indexed by the ego index.
\end{enumerate}

\paragraph{Actor.}
The actor $\pi_\phi$ maps the ego embedding to a distribution over valid phases. Raw logits are produced by a linear head, then invalid phases are suppressed by setting their logits to $-\infty$ before the softmax:
\[
  \mathbf{l} = \pi_\phi(\mathbf{h}_{\mathrm{ego}}^{\prime}),
  \qquad
  \tilde{l}_a = l_a + (1 - m_a)\cdot(-\infty),
  \qquad
  \pi(a \mid o) = \operatorname{softmax}(\tilde{\mathbf{l}}),
\]
where $\mathbf{l} \in \mathbb{R}^{A_{\max}}$ are the pre-mask logits and $m_a \in \{0,1\}$ is the per-phase validity flag. In the implementation $-\infty$ is approximated as $-10^9$. Appendix~\ref{sec:Appendix_masked_softmax} proves formally that masked phases receive zero probability and zero gradient.

\paragraph{Centralized joint-action critic.}
The critic operates on the full graph tensor and an estimate of all agents' action probabilities, following the CTDE design. For each candidate ego action $a$, the critic constructs a context vector by concatenating the flattened graph tensor $\mathrm{vec}(\mathbf{H}^{\prime})$, the matrix of all agents' action-probability vectors with the ego's row replaced by the one-hot vector for $a$, the ego embedding $\mathbf{h}_{\mathrm{ego}}^{\prime}$, and a one-hot encoding of the ego index:
\[
  Q_\theta(a \,|\, \mathbf{H}^{\prime}, \boldsymbol{\Pi}_{-i}, i) =
    \mathrm{MLP}_\theta\!\bigl[\,
      \mathrm{vec}(\mathbf{H}^{\prime})\;\Vert\;
      \mathrm{vec}(\boldsymbol{\Pi}_{a})\;\Vert\;
      \mathbf{h}_{\mathrm{ego}}^{\prime}\;\Vert\;
      \mathbf{e}_i\,
    \bigr],
\]
where $\boldsymbol{\Pi}_{a}$ substitutes the one-hot for $a$ into the ego row of the joint action-probability matrix $\boldsymbol{\Pi}$. Twin critics are maintained and the minimum is used for target computation as in standard SAC. The joint action structure requires $n$-step = 1 updates to keep the action-probability estimates on-policy. The input dimensionality of this critic and its implications for scalability at large $N$ are derived in Appendix~\ref{sec:Appendix_critic_complexity}.

\subsection{FGSv2: Action-Token Architecture}
\label{sec:Methodology_FGSv2}

FGSv2 preserves FRAP's phase-competition structure not as a final scalar score but as a set of intermediate \emph{action tokens}, one token per candidate phase. This changes the information topology of the architecture: rather than collapsing phase-level information into a single node embedding before graph communication and actor computation, the action tokens carry phase-specific features through to the actor head.

\paragraph{FRAP action-token encoder.}
The encoder is tapped at the phase-competition representation block of FRAP, before the $1{\times}1$ convolution that would otherwise produce a scalar pairwise-competition result. For action $a$ at intersection $i$, let $c_{i,a,b}$ denote the feature vector produced by the hidden competition layer for competitor $b$. The valid competitor set is
\[
  V_{i,a} = \{b \mid b \neq a,\; m_{i,b} = 1\}.
\]
If $V_{i,a}$ is non-empty, the competition features are pooled and projected into an action token; otherwise (only one valid phase) the token is set to zero:
\begin{equation}
  \mathbf{z}_{i,a} =
  \begin{cases}
    \operatorname{LayerNorm}\!\Bigl(
      \operatorname{ReLU}\!\Bigl(
        \operatorname{Adapter}\!\Bigl(
          \dfrac{1}{|V_{i,a}|}\sum_{b \in V_{i,a}} c_{i,a,b}
        \Bigr)
      \Bigr)
    \Bigr) & \text{if } m_{i,a}=1 \text{ and } |V_{i,a}|>0, \\[6pt]
    \mathbf{0} & \text{otherwise.}
  \end{cases}
  \label{eq:fgsv2_token}
\end{equation}
The full action-token tensor is $\mathbf{Z}_i \in \mathbb{R}^{A_{\max} \times d_z}$, with rows for invalid phases set to zero.

A node summary $\mathbf{s}_i \in \mathbb{R}^{d_z}$ aggregates information across all valid action tokens via masked mean-pooling followed by layer normalisation:
\begin{equation}
  \mathbf{s}_i =
    \operatorname{LayerNorm}\!\left(
      \frac{\displaystyle\sum_{a} m_{i,a}\,\mathbf{z}_{i,a}}{\displaystyle\max\!\left(\sum_{a} m_{i,a},\;1\right)}
    \right).
  \label{eq:fgsv2_summary}
\end{equation}
The $\max(\cdot, 1)$ guard prevents division by zero for intersections with no valid phase, which cannot occur during normal operation but ensures numerical safety during graph-padding.

\paragraph{Graph communication and residual gate.}
GATv2 operates on the node summaries rather than on the full action-token tensors, keeping communication overhead independent of the number of candidate phases. A layer normalisation is applied before the GATv2 input; let $\mathbf{u}_i = \operatorname{LayerNorm}(\mathbf{s}_i)$. The neighbourhood aggregation (self-loops included) is
\begin{equation}
  \tilde{\mathbf{g}}_i =
    \operatorname{ReLU}\!\left(
      \mathbf{W}_o\,\operatorname{GATv2Conv}\!\left(\{\mathbf{u}_j : j \in \mathcal{N}(i) \cup \{i\}\}\right)
    \right).
  \label{eq:fgsv2_gatv2}
\end{equation}
A learned scalar gate $\lambda$ (initialised to zero) controls the contribution of the graph context via an additive residual with layer normalisation:
\begin{equation}
  \mathbf{g}_i = \operatorname{LayerNorm}\!\left(\mathbf{s}_i + \lambda\,\tilde{\mathbf{g}}_i\right).
  \label{eq:fgsv2_gate}
\end{equation}
Because $\lambda = 0$ at initialisation, the network begins as a purely local policy; the gate learns to incorporate neighbour context only insofar as it reduces the SAC objective. Unlike a bounded gate ($\beta \in [0,1]$), the unconstrained scalar allows the network to amplify communication when it is beneficial.

\paragraph{Action-conditioned actor.}
For each candidate phase $a$, the actor concatenates the corresponding action token $\mathbf{z}_{i,a}$, the gated graph context $\mathbf{g}_i$ (broadcast across all phases), and a one-hot phase identity $\mathbf{e}_a$, before mapping through a two-layer MLP to a scalar logit:
\begin{equation}
  l_a = \operatorname{MLP}_\phi\!\bigl[\,\mathbf{z}_{i,a}\;\Vert\;\mathbf{g}_i\;\Vert\;\mathbf{e}_a\,\bigr].
  \label{eq:fgsv2_actor}
\end{equation}
Stacking over all $a$ gives $\mathbf{l} \in \mathbb{R}^{A_{\max}}$, which is masked and normalised identically to FGS (invalid logits set to $-10^9$, then softmax applied).

\paragraph{Centralized action-token critic.}
The critic extends the FGS design by incorporating the ego action token alongside the full graph context. For each candidate ego action $a$ of agent $k$, the critic input concatenates: the flattened stack of all agents' gated graph contexts $\operatorname{vec}(\mathbf{g}_1,\ldots,\mathbf{g}_N)$; the joint-action context $\mathbf{J}_k^{(a)}$ (the joint action-probability matrix with agent $k$'s row replaced by the one-hot for $a$, using hard replayed actions during TD updates and soft policy probabilities during the actor loss); the ego action token $\mathbf{z}_{k,a}$; the ego graph context $\mathbf{g}_k$; and a one-hot agent identity $\mathbf{e}_k$:
\begin{equation}
  Q_k(a) = f_Q\!\Bigl[\,
    \operatorname{vec}(\mathbf{g}_1,\ldots,\mathbf{g}_N)\;\Vert\;
    \operatorname{vec}(\mathbf{J}_k^{(a)})\;\Vert\;
    \mathbf{z}_{k,a}\;\Vert\;
    \mathbf{g}_k\;\Vert\;
    \mathbf{e}_k
  \,\Bigr].
  \label{eq:fgsv2_critic}
\end{equation}
Twin critics and $n$-step~$=1$ updates are retained from FGS. The use of hard previous actions from the replay buffer during critic updates and soft current policy probabilities during actor updates follows the MADDPG convention \citep{lowe2017multi} and is standard in off-policy CTDE methods.

\subsection{Training Algorithm}
\label{sec:Methodology_TrainingAlgorithm}

Algorithm~\ref{alg:fgs_training} presents the complete training procedure shared by all FGS and FGSv2 variants. All agents share a single set of actor and critic parameters, so the gradient updates in Lines~\ref{alg:line_critic}--\ref{alg:line_alpha} are computed jointly over all agent--step pairs in the sampled mini-batch. Algorithm~\ref{alg:fgsv2_forward} details the FGSv2 forward pass that maps a graph observation to policy logits and the graph contexts required by the centralised critic during training.

\begin{algorithm}[ht]
\caption{FGS / FGSv2 Training (shared procedure)}
\label{alg:fgs_training}
\begin{algorithmic}[1]
\Require SUMO scenario with $N$ intersections; episodes $E$; warm-up steps $T_{\text{warm}}$;
\Statex \quad batch size $B$; soft-update rate $\tau$; discount $\gamma$; target entropy $\mathcal{H}_{\text{target}}$
\State \textbf{Initialise} shared actor $\pi_\phi$; twin critics $Q_{\theta_1},Q_{\theta_2}$; target critics $\bar{Q}_j \leftarrow Q_{\theta_j}$, $j\in\{1,2\}$
\State \textbf{Initialise} log-temperature $\log\alpha$; residual gate $\lambda\leftarrow 0$ \textbf{(FGSv2 only)}
\State \textbf{Initialise} prioritised replay buffer $\mathcal{D}$ with capacity $C$; global step $t\leftarrow 0$
\For{episode $e = 1,\ldots,E$}
  \State Reset SUMO via TraCI; receive initial graph observation $o_0$
  \For{simulation step $t_{\text{ep}} = 1,\ldots,T_{\text{max}}$}
    \State $t \leftarrow t + 1$
    \If{$t < T_{\text{warm}}$} \hfill\(\triangleright\) \textit{warm-up: random actions to seed the buffer}
      \State Sample $a_i \sim \operatorname{Uniform}(\mathcal{A}_i)$ for all $i \in \mathcal{N}$
    \Else \hfill\(\triangleright\) \textit{on-policy action from learned policy}
      \State Compute $\pi_\phi(\cdot\mid o_{t-1})$ via Alg.~\ref{alg:fgsv2_forward}; sample $a_i \sim \pi_\phi$ for all $i$
    \EndIf
    \State Execute joint action $\mathbf{a} = (a_1,\ldots,a_N)$ in SUMO via TraCI
    \State Observe per-agent rewards $r_i \leftarrow W_{t-1}^{(i)} - W_t^{(i)}$ and next graph observation $o_t$
    \State Store transition $\bigl(o_{t-1},\,\mathbf{a},\,\mathbf{r},\,o_t,\,d_t\bigr)$ in $\mathcal{D}$ with maximum priority
    \If{$t \geq T_{\text{warm}}$} \hfill\(\triangleright\) \textit{one gradient update per environment step}
      \State Sample mini-batch $\mathcal{B} = \{(o,\mathbf{a},\mathbf{r},o',d)\}$ from $\mathcal{D}$ via PER
      \State Run forward pass (Alg.~\ref{alg:fgsv2_forward}) on $o$ and $o'$ for all samples in $\mathcal{B}$
      \State Compute soft target: $y_k \leftarrow r_k + \gamma(1-d)\,V_{\text{target}}(o')$ \hfill (Eqs.~\ref{eq:sac_target}--\ref{eq:sac_td_target})
      \State $\theta_j \leftarrow \theta_j - \eta_Q\,\nabla_{\!\theta_j}\mathcal{L}_Q(\theta_j)$, $j\in\{1,2\}$ \hfill (Eq.~\ref{eq:sac_critic_loss}) \label{alg:line_critic}
      \State $\phi \leftarrow \phi - \eta_\pi\,\nabla_\phi\,\mathcal{L}_\pi(\phi)$ \hfill (Eq.~\ref{eq:sac_actor_loss})
      \State $\log\alpha \leftarrow \log\alpha - \eta_\alpha\,\nabla_{\!\log\alpha}\,\mathcal{L}(\alpha)$ \hfill (Eq.~\ref{eq:sac_alpha_loss}) \label{alg:line_alpha}
      \State Soft-update targets: $\bar\theta_j \leftarrow \tau\theta_j + (1-\tau)\bar\theta_j$, $j\in\{1,2\}$
      \State Update PER priorities with TD errors $|Q_{\theta_1}(o,a_k) - y_k|$
    \EndIf
  \EndFor
  \If{$e \bmod 5 = 0$} \hfill\(\triangleright\) \textit{evaluation checkpoint every 5 episodes}
    \State Run one deterministic episode (greedy; exploration suppressed)
    \State Log MTT, MWT, MD, TP to Weights \& Biases; save checkpoint
  \EndIf
\EndFor
\end{algorithmic}
\end{algorithm}

\begin{algorithm}[ht]
\caption{FGSv2 Forward Pass}
\label{alg:fgsv2_forward}
\begin{algorithmic}[1]
\Require Graph observation $o = \{\mathbf{X},\mathcal{E},\mathbf{W},\mathbf{M}_{\text{pair}},\mathbf{M}_{\text{comp}},\mathbf{m},\mathbf{a}_{t-1}\}$; ego index $k$
\Ensure Policy distribution $\pi_k(\cdot\mid o)$; graph contexts $\{\mathbf{g}_i\}$ and action tokens $\{\mathbf{z}_{i,a}\}$ (training)
\Statex \rule{\linewidth}{0.3pt}
\Statex \textbf{Phase 1 — FRAP action-token encoding}
\ForAll{intersection $i \in \mathcal{N}$ \textbf{and} movement $u$}
  \State $\mathbf{e}_{i,u} \leftarrow \operatorname{ReLU}\!\bigl(W_{\text{lane}}\bigl[\sigma(\operatorname{Emb}_{\text{phase}}(p_{i,u})),\;\sigma(W_d\,d_{i,u})\bigr]\bigr)$
  \hfill\(\triangleright\) \textit{lane embedding}
\EndFor
\ForAll{$i \in \mathcal{N}$, valid phase $a$ ($m_{i,a}=1$)}
  \State $\boldsymbol{\phi}_{i,a} \leftarrow \sum_u w_{i,a,u}\,\mathbf{e}_{i,u}$
  \hfill\(\triangleright\) \textit{phase-pair embedding (scaled sum over movements)}
  \ForAll{competitor $b \in V_{i,a}$}
    \State $\mathbf{r}_{i,a,b} \leftarrow \operatorname{ReLU}(\operatorname{Conv}_{\text{rel}}(\operatorname{Emb}_{\text{rel}}(R_{i,a,b})))$
    \State $\mathbf{c}_{i,a,b} \leftarrow \operatorname{ReLU}\!\bigl(\operatorname{Conv}_{\text{hid}}([\boldsymbol{\phi}_{i,a},\boldsymbol{\phi}_{i,b}] \odot \mathbf{r}_{i,a,b})\bigr)$
    \hfill\(\triangleright\) \textit{competition feature}
  \EndFor
  \If{$|V_{i,a}| > 0$}
    \State $\mathbf{z}_{i,a} \leftarrow \operatorname{LayerNorm}\!\bigl(\operatorname{ReLU}\!\bigl(\operatorname{Adapter}(\tfrac{1}{|V_{i,a}|}\textstyle\sum_{b} \mathbf{c}_{i,a,b})\bigr)\bigr)$ \hfill (Eq.~\ref{eq:fgsv2_token})
  \Else
    \State $\mathbf{z}_{i,a} \leftarrow \mathbf{0}$ \hfill\(\triangleright\) \textit{no competitors: single valid phase}
  \EndIf
\EndFor
\State Set $\mathbf{z}_{i,a} \leftarrow \mathbf{0}$ for all invalid phases ($m_{i,a}=0$)
\Statex \rule{\linewidth}{0.3pt}
\Statex \textbf{Phase 2 — Node summary}
\ForAll{$i \in \mathcal{N}$}
  \State $\mathbf{s}_i \leftarrow \operatorname{LayerNorm}\!\bigl(\textstyle\sum_a m_{i,a}\,\mathbf{z}_{i,a} \;/\; \max(\textstyle\sum_a m_{i,a},1)\bigr)$ \hfill (Eq.~\ref{eq:fgsv2_summary})
\EndFor
\Statex \rule{\linewidth}{0.3pt}
\Statex \textbf{Phase 3 — GATv2 communication with residual gate}
\ForAll{$i \in \mathcal{N}$}
  \State $\mathbf{u}_i \leftarrow \operatorname{LayerNorm}(\mathbf{s}_i)$
  \hfill\(\triangleright\) \textit{pre-normalise before attention}
  \State $\tilde{\mathbf{g}}_i \leftarrow \operatorname{ReLU}\!\bigl(\mathbf{W}_o\;\operatorname{GATv2Conv}(\{\mathbf{u}_j : j \in \mathcal{N}(i)\cup\{i\}\})\bigr)$
  \hfill (Eq.~\ref{eq:fgsv2_gatv2})
  \State $\mathbf{g}_i \leftarrow \operatorname{LayerNorm}(\mathbf{s}_i + \lambda\,\tilde{\mathbf{g}}_i)$
  \hfill (Eq.~\ref{eq:fgsv2_gate})
\EndFor
\Statex \rule{\linewidth}{0.3pt}
\Statex \textbf{Phase 4 — Action-conditioned actor for ego agent $k$}
\ForAll{phase $a = 1,\ldots,A_{\max}$}
  \If{$m_{k,a} = 1$}
    \State $l_a \leftarrow \operatorname{MLP}_\phi[\mathbf{z}_{k,a}\;\Vert\;\mathbf{g}_k\;\Vert\;\mathbf{e}_a]$
    \hfill (Eq.~\ref{eq:fgsv2_actor})
  \Else
    \State $l_a \leftarrow -10^9$
    \hfill\(\triangleright\) \textit{suppress invalid phases before softmax}
  \EndIf
\EndFor
\State \Return $\pi_k(\cdot\mid o) = \operatorname{softmax}(\mathbf{l}_k)$; also $\{\mathbf{z}_{i,a}\},\{\mathbf{g}_i\}$ for critic (Eq.~\ref{eq:fgsv2_critic})
\end{algorithmic}
\end{algorithm}
