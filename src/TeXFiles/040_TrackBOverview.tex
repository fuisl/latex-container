\section{Business Capability and Journeys}
\label{sec:TrackB_BusinessCapability}
%=============================================================================

The platform turns natural-language requirements into \gls{qa} artifacts with
explicit human review gates.

Target users:
\begin{itemize}
    \item QA engineers: accelerate test creation with review/edit control.
    \item Product engineers/managers: translate requirements into structured outputs.
    \item Platform/ops: operate a multi-tenant, governed, observable service.
\end{itemize}

Journeys (human-in-the-loop):
\begin{itemize}
    \item Requirement $\rightarrow$ Test cases: generate $N$ candidates, pause for review; approve/edit/regenerate.
    \item Test case $\rightarrow$ Test steps: generate ordered steps, pause for edits; approve/reject with feedback.
\end{itemize}

\section{Key Learning Points}
\label{sec:TrackB_Learnings}

\begin{itemize}
    \item Human-in-the-loop gates materially improve correctness versus one-shot generation.
    \item State-machine orchestration yields predictable behavior and easier recovery.
    \item Governance and observability must be designed upfront to avoid rework.
    \item Managed deployment (Cloud Run) reduces ops overhead at the cost of fine-grained control.
\end{itemize}

These learnings informed design choices across the platform and guided trade-offs between flexibility and control.
