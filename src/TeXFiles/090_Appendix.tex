\chapter{Tools and Technologies}
\label{ch:Appendix_ToolsAndTechnologies}

The FGS experimental pipeline integrates the following software components.
\textbf{SUMO 1.18} \citep{lopez2018microscopic} provides microscopic traffic simulation and exposes the TraCI socket interface for per-step observation and action exchange.
\textbf{SUMO-RL} \citep{alegre2021sumorl} wraps SUMO environments as Gymnasium-compatible multi-agent environments; graph observation wrappers and custom reward functions were added for this work.
\textbf{RLlib 2.x} (Ray) provides the distributed multi-agent training loop; custom policy modules subclassing \texttt{TorchRLModule} implement the FGS actor, critic, and encoder components.
\textbf{PyTorch Geometric} implements the GATv2 convolution (\texttt{GATv2Conv}) and heterogeneous graph batching utilities.
\textbf{Hydra} structures all experiment configurations as composable YAML files; scenario, algorithm, and architecture parameters are varied without code changes.
\textbf{Weights \& Biases} \citep{wandb} tracks all training runs, logs per-episode RESCO metrics, and stores model checkpoints.

\chapter{Losses and Network Objectives}
\label{ch:Appendix_LossesObjectives}

The full discrete-action SAC objective functions are given in Section~\ref{sec:Methodology_SAC}.
The PPO clipped objective is $\mathcal{L}^{\text{CLIP}}(\phi) = \mathbb{E}_t\!\left[\min\!\left(r_t(\phi)\hat{A}_t,\, \mathrm{clip}(r_t(\phi), 1-\epsilon, 1+\epsilon)\hat{A}_t\right)\right]$ where $r_t(\phi) = \pi_\phi(a_t|s_t)/\pi_{\phi_\text{old}}(a_t|s_t)$ and $\hat{A}_t$ is the generalised advantage estimate with $\lambda=0.95$ and $\gamma=0.99$.
The joint loss for PPO includes value function and entropy terms: $\mathcal{L} = \mathcal{L}^{\text{CLIP}} - c_1 \mathcal{L}^{VF} + c_2 \mathcal{H}[\pi_\phi]$ with $c_1=0.5$ and $c_2=10^{-3}$.

\chapter{Full Experimental Results}
\label{ch:Appendix_FullResults}

This appendix presents complete numerical results across all six German RESCO scenarios and all evaluated methods, formatted after the style of the original RESCO supplementary material \citep{ault2021reinforcement}. Scenarios are classified into three scale tiers following the RESCO benchmark nomenclature:
\begin{itemize}
  \item \textbf{Single-intersection} (\textit{Single}): ingolstadt1 ($N=1$) and cologne1 ($N=1$).
  \item \textbf{Corridor} (\textit{Corr.}): ingolstadt7 ($N=7$, arterial) and cologne3 ($N=3$, arterial).
  \item \textbf{Regional} (\textit{Reg.}): ingolstadt21 ($N=21$) and cologne8 ($N=8$).
\end{itemize}

All results from this work are \textit{best-validation} values: the validation checkpoint with minimum mean waiting time observed during training, evaluated on a fixed held-out demand seed. Results marked $\circ$ indicate scenarios where the method was not yet run or crashed before producing valid metrics; these cells are candidates for re-run. Results marked $\dagger$ are from a partially trained run (FGS(MLP+GATv2+SAC) on Ing.~Reg.\ terminated after 17 of 200 training episodes due to resource exhaustion) and should not be compared directly to converged results. RESCO reference values (IDQN, IPPO, FMA2C) are reproduced from the original paper \citep{ault2021reinforcement} and represent best-episode averages over five random seeds.
